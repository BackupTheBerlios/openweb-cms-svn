Détails d'un document

Cette page présente le détail des informations et les actions disponibles pour un document existant.

Le type du document indique la partie du site dans laquelle le document se trouvera (le plus souvent dans la section article ou intro).

Le champ auteurs indique le noms des auteurs tels qu'ils sont indiqués dans la source du document (il ne correspond pas nécessairement à l'utilisateur du backend qui a pris en charge sa publication). De même, les date de publication et date de modification sont celles qui sont indiqués dans la source et non les dates d'ajout ou de modification dans le backend (cela permet de corriger une erreur dans un document sans le faire apparaître dans la section Derniers articles de la page d'accueil).

La section Formats disponibles permet de consulter le document dans tous les formats disponibles depuis le backend, et éventuellement depuis le site si l'article est en-ligne. Un simple rédacteur ne peut pas consulter un document dans l'état temporaire si ce dernier a été ajouté par un autre rédacteur : c'est le principe de l'état temporaire, qui permet au rédacteur de préparer son document avant de le soumettre à tout le monde. En revanche, un rédacteur en chef a le droit de faire ce qu'il veut avec le document.

Selon les permissions que vous avez sur le document, la section Fichiers annexes permet :

  • d'ajouter des fichiers annexes ;
  • de les supprimer ;
  • de les consulter.

La publication d'un document suit trois étapes :

  1. le document est téléchargé sur le backend par le rédacteur, il est dans l'état temporaire : le rédacteur est le seul à pouvoir le consulter, il vérifie que l'upload s'est déroulé correctement et ajoute les annexes ;
  2. le rédacteur passe alors le document de l'état temporaire en hors-ligne : tout le monde peut relire le document et le contrôle de la publication passe alors aux mains des rédacteurs en chef, qui sont les seuls à pouvoir modifier le document. Un rédacteur en chef peut décider qu'un document n'est pas encore prêt et le remettre en temporaire pour permettre au rédacteur d'apporter ses modifications ;
  3. quand le document est relu et prêt à être publié, le rédacteur en chef peut mettre en-ligne le document.
